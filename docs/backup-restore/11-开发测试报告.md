## 4. API 变更验证

- 六个业务 CRD 保留必填 `clusterRef`，独立范围对象已删除，不提供旧模型兼容。
- 选择配置合并到 `BackupPolicy.spec.selection`；API 类型、Webhook、Controller、CRD base、Helm CRD、部署 bundle、样例、测试和文档已同步更新。
- `BackupTask` 只接受必填 `policyRef` 作为业务入口，Controller 在执行前一次性固化 `selectionSnapshot`、Repository UID 与 Policy generation。
- 旧字段请求触发 strict decoding error。
- 缺少 `clusterRef` 的请求触发 Schema 校验错误。
- `PVCSelectionSpec.enabled` 改为显式序列化、安全默认 false；回归测试和集群任务均确认 false 不会在冻结后变为 true。

## 6. 未通过或未执行项

- 当前集群没有 CSI Snapshot API、Snapshot Controller、支持快照的 CSI Driver 和 VolumeSnapshotClass，PVC Snapshot/Restore 未执行。
- 未部署独立外部 SFTP 服务；SFTP Adapter 的嵌入式单元测试通过，但不替代外部 SFTP E2E。
- 未执行 10 万对象、100 PVC、大于 100GiB 归档或网络抖动压力测试。
- Kubernetes v1.36.1 不在当前正式兼容矩阵 v1.28-v1.32 内。
- 当前已提供内置管理员 WebUI；独立平台 REST API 仍为后续能力，当前 WebUI 通过内置受控 API 访问 Kubernetes CRD。

## 7. 发布风险

1. Local Repo 是单节点 hostPath，不是异地灾备。
2. Restore 逐对象 checkpoint、自动重试和 Rename 重入幂等仍需增强。
3. CRD 与 Custom Resource 恢复尚未拆分为“等待 CRD Established 后再刷新 Mapper”的独立阶段。
4. ClusterRole 权限较高，只能授予 Operator ServiceAccount，普通用户禁止模拟该账号。
5. 当前为集群管理员单租户模式；未来多租户必须由外部 ACL/API 和 Namespace 执行时复验实现。

因此当前结论是：Docker Desktop 上不含 PVC 的本地 MVP 已完整部署并通过真实备份恢复验收，但不能据此声明 PVC、外部 SFTP、大规模或生产级灾备能力已认证。
