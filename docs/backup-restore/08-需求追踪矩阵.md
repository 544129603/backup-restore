# 容器平台备份与恢复插件：需求追踪矩阵与设计自检

## 1. 追踪规则

- 功能需求权威编号在《01-需求规格说明书》；页面、CRD、控制器、API、测试均必须引用稳定编号。
- 一个需求可以对应多个实现/用例；矩阵中至少给出主责任对象，避免“所有 Controller/所有页面”的不可执行映射。
- 版本变更不得删除既有验收映射；延期必须同时更新需求版本、页面开关、API admission 和测试期望。

业务目标缩写：G1=建立可靠副本；G2=范围与租户隔离；G3=自动化 RPO；G4=可观测执行；G5=确定性恢复；G6=资产生命周期与安全。

## 6. 最终设计自检

| 检查项 | 结论 | 证据/剩余动作 |
|---|---|---|
| 1. 六个核心 CRD 是否职责重叠 | 无实质重叠 | Repo=位置、Scope=内容、Policy=时间、Task=执行、Record=资产、Restore=恢复；Config 仅默认值 |
| 2. CRD 是否全部 Cluster-scoped | 是 | 7 个 kind 统一 scope；示例均无 metadata.namespace |
| 3. 多租户是否存在越权风险 | 有结构性风险但已给纵深控制 | Cluster scope 不能原生行级 RBAC；强制平台 API、租户无 list/watch、Webhook+执行时复验；上线前需渗透测试 |
| 4. Record 是否受业务 Namespace 生命周期影响 | 否 | 无 namespaced ownerRef；包、Record、Snapshot 由自身 finalizer 管理 |
| 5. Repo Secret 引用是否安全 | 设计满足 | namespace/name/key，白名单，Secret data 不入 status/log；仍需验证平台后端响应脱敏 |
| 6. 定时任务是否防重/补偿 | 是 | policyUID/scheduledTime UTC；Skip/RunOnce/RunAll；控制器重启从 status+Task 索引计算 |
| 7. BackupTask 是否幂等/故障恢复 | 是 | 确定名称、步骤 checkpoint、外部采纳、heartbeat、worker 重建 |
| 8. Record 是否仅完整上传后可用 | 是 | staging→`.done`→读回 verify→Record；验收有中断/篡改 |
| 9. PVC 快照生命周期是否清晰 | 是 | Retain/DeleteWithRecord；Task 不拥有已提交快照；Record 三种删除 mode |
| 10. 恢复顺序是否覆盖依赖 | 是 | Namespace→CRD→集群依赖→RBAC/配置→PVC→Service→Workload→CR；owner 二次 patch |
| 11. 冲突策略是否明确 | 基本明确 | Skip/Fail 完整；Overwrite/Rename 已限类型；MergePatch 延期。需研发前冻结“可重建白名单” |
| 12. 页面是否覆盖正常/异常/空/权限 | 是 | 02 全局规范+每页表格/向导 |
| 13. 每个需求是否有验收条件 | 是 | 01 每需求行+07 独立 GWT |
| 14. 每需求是否追踪 CRD/Controller/API/页面 | 是 | 本矩阵 22/22 主需求均映射 |
| 15. MVP 是否过度设计 | 有两处需控制 | Hook schema但关闭、Config 可裁剪 UI；Worker Job/全量校验/权限纵深不可裁剪。DryRun/SC复杂映射已延期 |
| 16. V1 不支持能力是否清楚 | 是 | 跨集群、文件级、对象存储、App Hook、Repo迁移、MergePatch、完整 DryRun 均明确延期 |

## 7. 主要风险清单

| 风险 | 概率/影响 | 早期信号 | 缓解/决策门 |
|---|---|---|---|
| Cluster-scoped CRD 被直接授予租户 list/watch | 中/严重 | 权限审计发现 tenant ClusterRole | 安装器拒绝危险绑定；定期 RBAC scanner；平台 API 唯一入口；安全评审门 |
| CSI 厂商行为差异/handle 不可复用到新 NS | 高/高 | VSC Ready 但目标 PVC 失败 | 认证矩阵；驱动 capability profile；不支持时只允许原 NS/元数据；发布门覆盖目标厂商 |
| SFTP 吞吐/原子 rename/容量扩展差异 | 中/高 | capability check 缺失、长尾超时 | `.done` 屏障；statvfs optional；流式/限流；规模超门限推荐对象存储版本 |
| Local Repo 单点被误当灾备 | 高/高 | 只有 HostPath 且合规标绿 | UI 强风险标识、合规报告区分故障域、生产建议 SFTP/后续对象存储 |
| Overwrite/重建造成业务中断或数据丢失 | 中/严重 | 大量 immutable conflict | 默认 Skip；高危类型禁用；二次确认、Diff、白名单；PVC 永不自动重建 |
| Secret 包泄露/密钥丢失 | 低/严重 | 未加密 Scope 包含 Secret；key 无备份 | 强制加密、envelope/key version、密钥轮换/托管、下载鉴权；密钥恢复手册 |
| Kubernetes 资源采集非跨 GVR 原子 | 高/中 | 采集窗口资源频繁变化 | 明示 BestEffort；稳定排序/RV/警告；应用一致性留 Hook；关键数据库依赖快照/逻辑备份 |
| Record CR 与 Repo/快照事实漂移 | 中/高 | RepoUnavailable/SnapshotMissing 增多 | 周期 verify、finalizer、GC 只删有 ownership、审计外部删除 |
| CRD conversion/admission webhook 阻止恢复 | 中/高 | precheck dry-run 失败 | CRD/依赖先恢复、webhook connectivity 检查、对象级错误和手工处理指南 |
| status 过大/API Server 写放大 | 中/中 | etcd object 逼近 1.5MiB、QPS 高 | errors最多100、细节外置、2s/1%节流、摘要字段、压测门 |

## 9. 推荐 MVP 剪裁边界

必须保留：两种 Repo、范围预览、CSI 预检、Cron 防重补偿、手动/取消/重试、Task/Record 分离、`.done`+全量 hash、同集群原位/新 Namespace、Skip/Fail 与受限 Overwrite、三模式删除、平台授权/RBAC/Webhook、日志/指标/告警。

不进入 V1.0 执行能力：Hook/AppConsistent、文件级数据、跨集群、S3/OBS/MinIO、自动 Repo 迁移、通用 MergePatch、任意 CR Rename、任意资源 delete/recreate、全对象 server-side DryRun。CRD 中仅保留不会迫使实现复杂执行器的向后兼容字段，并由 admission 明确拒绝启用。
